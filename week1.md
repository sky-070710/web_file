  ### HTML - 뼈대

HTML은 HyperText Markup Language의 약자로 마크업 언어이다.

- 마크업 언어 : 문서의 형식이나 데이터의 구조를 명시하기 위한 규칙들을 정의한 컴퓨터 언어의 일종

HTML은 태그( <> )라는 것으로 구성이되는데, 이 태그 안에 명령어를 넣어 명시를 해주는 역할을 가진다.

- 계층 구조적 형식을 가진다는 점(큰 박스 안에 작은 박스가 들어가는 중첩 구조를 가진다)
- 모든 브라우저가 기본적으로 이해한다는 특징을 가지고 있다.

### CSS - 디자인

CSS는 HTML로 만든 문자 구조를 꾸며주는 언어이다. 

위 HTML에 소개할 때 허전해 보이는 HTML문장을 CSS에 연결하여 예쁘게 꾸며줄 수 있다.

특징

- 스타일 시트 언어의 형태를 가진다
    - 스타일 시트 :  어떤 요소를 어떻게 꾸밀지 정해놓은 모음집이라고 생각하면 편하다.
- HTML과 분리 되어 있어 따로 관리할 수 있다(유지보수가 편해짐)
- 케스케이딩 형태를 띄게 된다.
    - 스타일이 여러 곳에서 정의된다고 가정하면, 우선순위 규칙에 따라 합쳐져서 적용되는 계단식 구조를 가진다.
- 오버라이딩의 특징을 가진다
    - 동일한 CSS 속성에 대해 다른 값이 설정될 경우, 우선순위가 높은 스타일이 적용된다.
    - 우선 순위 기준(어디에 작성되어 있는가, 얼마나 구체적으로 명시하고 지목했는가, 누가 나중에 적혔는가에 대한 작성 순서)를 판단하여 우선순위를 결정하게 된다

### JS - 웹의 동작

JavaScript는 웹페이지에 동작 부분을 담당한다. HTML로 뼈대를 구성하고, CSS로 디자인을 만들어준뒤, JS로 여러 기능들을 만들어서 동작한다고 생각하면 된다.

ex) 버튼을 클릭 하였을 때, 반응하고, 데이터를 계산하고, 화면에 실시간으로 값을 보여주는 일 들을 담당한다고 생각하면 편하다.

특징 

- 프로그래밍 언어이기 때문에 변수, 조건, 반복 같은 요소들을 가지고 있다.
- 동적의 특징을 가져서 페이지를 새로고침 하지 않아도 실시간으로 내용을 바꿀 수 있다.
- 인터프리터 언어로 미리 컴퍼일하지 않아도 브라우저가 한 줄씩 읽으며 실행된다.
    - 인터프리터 : 한 줄씩 코드가 실행
    - 컴파일러 : 한 번에 기계어로 번역하여 실행

### HTML, CSS, JS의 차이점

| --- | --- | --- | --- |
| 언어 종류 | 마크업 언어 | 스타일 시트 언어  | 프로그래밍 언어 |
| 역할 | 구조  | 디자인  | 동작 |
| 정적 / 동적 | 정적 | 정적 | 동적 |

### HTML, CSS, JS의 연결점

이 세개의 언어는 하나의 웹 페이지를 만들기 위한 협업과 같은 관계를 가진다.

**HTML이 기준점**

CSS와 JS는 모두 HTML을 기반으로 작동하게 된다.

- CSS는 HTML에 작성된 태그를 보고 스타일 하는 일을 한다.
- JS는 HTML에 작성된 요소를 보고 구조를 읽고 조작하게 된다.

그리고 브라우저는 세개의 언어를 한 번에 처리해서 보여주는게 아닌 하나씩 실행하여서 사용자에게 보여주게 된다.

1. HTML을 읽고 페이지의 구조를 만든다
2. CSS를 적용하여 HTML의 화면을 꾸미고 다듬는다
3. JS를 실행하여 페이지에 동작을 부여하게 된다.

### 궁금증

1. 왜 이렇게 HTML, CSS, JS 같이 3개의 언어로 나뉘게 되었을까? 하나로 처리하면 안되나??
    - 왜 그러는지 한 마디로 설명하면 유지보수성이 높다는 점 때문이다.
    - 내용 → HTML, 디자인 → CSS, 동작 → JS가 담당하는 구조로서 내용을 바꾸고 싶으면 HTML 수정, 디자인을 바꾸고 싶으면 CSS, 동작을 바꾸고 싶으면 JS를 변경하면 되지만,
    - 하나의 언어로 모두 연결되어 있으면 코드 전체를 바꾸어야 한다는 불상사가 생기기 때문에 나눠서 유지보수성을 높히고 있다.
2. 꼭 HTML, CSS, JS여야한 할까? 다른 언어로 대체할 수 있을까?완전히 대체할 수 있는 언어는 사실상 없다. 하지만 부분적으로 대체하거나, 변형해서 쓰는 기술들은 많이 있다.
    - 왜 어려운가!
        - 브라우저는 일단 HTML, CSS, JS만 이해하기 때문이다. 
        브라우저는 일종의 실행 환경인데, 이 환경 안에서 동작하도록 만들어진 언어가 HTML, CSS, JS기 때문이다.
    - 그럼 아예 못하나??
        - 다른 언어로 짠 다음, 결과적으로 HTML·CSS·JS로 변환하면 된다.
        - ex) Pug, HAML 같은 언어는 HTML보다 더 짧고 깔끔하게 쓸 수 있어요. 하지만 결국 HTML로 변환되어 브라우저에 전달
        ex) Markdown도 비슷해요. 우리가 노션이나 깃허브에서 쓰는 그 마크다운인데, 결국 HTML로 바뀌어 화면에 나오게 된다.
        
        * 최근에는 웹 어셈블리(C, C++, Rust, Go 같은 언어로 짠 코드를 브라우저에서 직접 실행할 수 있게 해주는 기술) 라는 기술이 나와 JS에 대한 어느정도 기술이 대체가 가능하다고 하지만 완전하지는 않다고 한다.
